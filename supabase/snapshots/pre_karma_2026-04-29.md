## Rollback procedure

The Living Eamon database schema is fully reproducible from `supabase/migrations/*.sql`. To roll back to this point:

1. Revert any new migrations introduced after this snapshot.
2. `git checkout d6d2262 -- supabase/migrations/` (or `git checkout dev` if the broken commits are isolated to a branch).
3. Re-run `./scripts/db-push.sh dev` (or prod) — migrations are idempotent on a clean target.

## Why a manifest, not a `pg_dump`

This project has no local Supabase CLI; migrations are pushed via the Management API (`scripts/db-push.sh`). The migration files themselves are the canonical schema source of truth.
